# 마케팅잡픽 배포 가이드 — 전체 개요

## 어떤 시스템을 만드나요?

```
┌──────────────────┐
│  사용자 브라우저  │  ← 정적 HTML (Vercel CDN에서 서빙)
└────────┬─────────┘
         │ fetch
         ↓
┌──────────────────┐    ┌─────────────────────────┐
│  Vercel API      │    │  매시간 cron (정리·검증)  │
│  /api/jobs       │    │  /api/cron-sync         │
│  /api/feedback   │    │  - 마감 지난 공고 비활성  │
│  /api/users      │    │  - 제보 3+ 자동 검증      │
└────────┬─────────┘    └──────────┬──────────────┘
         ↓                         ↓
┌────────────────────────────────────┐
│  Supabase (PostgreSQL)             │
│  ├ users (가입자)                  │
│  ├ jobs (공고 - 시드 17건 + 사용자 제보) │
│  ├ feedback (피드백)               │
│  ├ sync_logs (cron 로그)           │
│  └ admins (관리자)                 │
└────────────────────────────────────┘
```

## 단계별 진행 순서

워크넷 인증키 발급은 **신청만 백그라운드로 해두고** 사이트는 먼저 띄웁니다.
워크넷 없이도 시드 데이터 17건 + 사용자 제보로 작동해요.

| 단계 | 내용 | 사용자 작업 시간 | 필수/선택 |
|---|---|---|---|
| ~~1~~ | ~~워크넷 API~~ | (이미 신청 완료, 승인 대기) | 나중 |
| 2 | GitHub 가입 + 저장소 | 15분 | **필수** |
| 3 | Supabase 프로젝트 + DB | 30분 | **필수** |
| 4 | Vercel 배포 | 30분 | **필수** |
| 5 | 도메인 연결 | 30분 | 선택 |
| 6 | 프론트 연동 (Claude에게 부탁) | 5분 | **필수** |
| 7 | 관리자 등록 | 5분 | **필수** |

**합계**: 사용자 실작업 약 2시간으로 **오늘 안에 배포 가능**

## 비용

- **무료 옵션**: Supabase Free + Vercel Hobby + 무료 vercel.app 도메인 = **0원**
  - Vercel Hobby cron은 하루 2번만 실행됨 → 외부 무료 cron(cron-job.org)으로 우회 가능
- **권장 옵션**: Supabase Free + Vercel Pro($20/월) = **월 약 2.5만원**
- 사용자 1만명 넘으면 Supabase Pro($25/월) 추가

## 워크넷은 나중에 어떻게 추가하나요?

승인 떨어지면 **두 가지 작업만** 추가하면 돼요:
1. Vercel 환경변수에 `WORKNET_API_KEY` 추가
2. `api/cron-sync.js`에 워크넷 호출 코드 추가 (제가 다시 작성해드림)

지금 만든 사이트는 그대로 살아 있고, 데이터 소스만 늘어나는 구조예요.
