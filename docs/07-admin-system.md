# 7단계: 관리자 시스템 사용 가이드

## 관리자가 할 수 있는 일

### ① 부적절한 공고 삭제
- 관리자 모드 진입 시 모든 공고 카드에 🗑 *삭제* 버튼 등장
- 클릭 → 삭제 사유 입력 → 확인 → DB에서 soft-delete (is_deleted=true)
- 일반 사용자에게는 더 이상 보이지 않음
- 잘못 삭제했을 때는 Supabase Table Editor에서 `is_deleted` 를 false로 되돌리면 복구 가능

### ② 가입 사용자 정보 검토
- *관리자 대시보드* 탭 → 사용자 목록 표시
- 가입일, 이메일, 학력, 관심 직무, 보유 스킬, 자격증 모두 확인 가능
- CSV 다운로드 가능

### ③ 피드백 상태 변경
- 오류 리포트 탭에서 각 피드백의 상태를 *검토 대기* → *확인 중* → *해결됨* 변경
- 대응한 피드백을 마킹해서 사용자에게 알림

## 관리자 등록 방법

### 본인을 관리자로 만들기
1. Supabase 대시보드 → Table Editor → `admins` 테이블
2. **+ Insert row** 클릭
3. 입력:
   - email: 본인 이메일 (관리자 모드 진입할 때 사용)
   - name: 본인 이름
   - role: `super_admin`
4. Save 클릭

### 다른 사람을 관리자로 추가
같은 방식으로 새 row 추가, role은:
- `super_admin` — 모든 권한
- `admin` — 공고 삭제, 피드백 상태 변경
- `viewer` — 읽기 전용 (사용자 정보 조회만)

## 관리자 모드 진입 방법

1. 사이트 우측 상단 프로필 칩 클릭 → 프로필 편집 모달 열림
2. 모달 하단 **🔐 관리자 모드 진입** 버튼 클릭
3. 등록된 관리자 이메일 입력
4. 시스템이 admins 테이블 조회 → 일치하면 모드 활성화
5. 사이트 상단에 **빨간 관리자 배너** 표시
6. 모든 공고 카드에 🗑 버튼, 새 *관리자 대시보드* 탭 노출

## 보안 주의사항

⚠️ **이 시스템은 학습·시연 단계의 보안 수준입니다.**
실제 서비스에서는 다음을 추가 권장:

1. **이메일 OTP 인증** — 입력한 이메일로 코드 보내서 본인 확인 (Supabase Auth 사용)
2. **2FA (이중 인증)** — 추가 보안 단계
3. **감사 로그** — 누가 언제 어떤 공고를 삭제했는지 기록 (이미 deleted_by, deleted_at 필드는 있음)

지금 단계에서는 이메일만 입력하면 관리자 모드가 활성화돼요.
*"우리 사이트 안에 관리자가 본인뿐이고, 본인 이메일을 다른 사람이 모를 것"* 이라는 가정이 깔려 있어요.
